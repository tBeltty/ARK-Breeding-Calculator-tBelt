# Definition of Done (Technical)

## Purpose

The Definition of Done ensures consistent quality and prevents incomplete work from being merged.

---

## Mandatory Conditions

A task is considered done only if:

1. Code follows Architecture Guidelines
2. Code follows Coding Standards
3. Tests are written and meaningful
4. Coverage thresholds are respected
5. Performance is not degraded
6. No new technical debt is introduced

---

## Documentation

1. ADR created if required
2. Public APIs documented
3. Non obvious behavior explained

---

## Review Requirements

1. Code reviewed
2. Tests reviewed
3. Architectural impact assessed

---

## Final Rule

If any condition is unmet, the task is not done.
